Node.js vs A Browser

Node.js and a browser are both able to execute JavaScript, which makes them have several things in common.
They are of course not the same and here will be covered some differences between Node.js and a browser.

JavaScript Runtime Enviornment

Node.js IS a JavaScript Runtime Enviornment and a browser HAS a JavaScript Runtime Enviornment.
Each browser can have its own JavaScript Runtime Enviornment.

Most JavaScript Runtime Enviornments have the following:
  • A JavaScript Engine
  • A Callback Queue
  • An Event Loop
  • API's
Node.js and all of the most popular browsers have all of theese four elements.
The implementation of each element is different between the enviornments.

JavaScript Engine

Here are some engines of the some of the most popular browsers:
  • V8 (Google Chrome & Microsoft Edge)
  • JavaScriptCore (Apple Safari)
  • SpiderMonkey (Mozilla Firefox)
  • Carakan (Opera)
  • Chakra (Microsoft Internet Explorer)
Node.js uses the V8 engine.

All JavaScript engines have these two elements:
  • The Call Stack
  • The Memmory Heap
The JavaScript engines only run one thread of execution.

The Callback Queue

This holds callbacks to asyncronous operations.
Callbacks are added to the callback queue by using API's.

The Event Loop

Continually checks if the call stack is empty.
If the call stack is empty, it can push a function from the callback queue on to the call stack.

API's

Node.js and the most popular browsers have some API's in common, but they are implemented differently.

Here are some API's they have in common:
  • The setTimeout and clearTimeout functions
  • The setInterval and clearClearInterval functions
