Express - Create a Route

This is a guide on how to make routes for an express server or router.
To see how to make an express server, go to this guide: Create a Server.
To see how to make an express router, go to this guide: Router.
The endpoint of a route will be the domain followed by the path.
For example the endpoint could be: "localhost:3000/about" where:

  • "localhost:3000" is the domain
  • "/about" is the path
When you go to a web-page, the HTTP get-method is used.

Here is the guide on, how to make a route:
  1. Use the a method on the server (or router) object to define a route for that HTTP method
    (Here the get method is used on the server object "app", to make a route for an HTTP get-method):

    app.get();


    (Server and router objects has methods for several HTTP methods such as: get, post, put, patch and delete).
  2. Define the the path for the route by adding it as the first argument
    (In this case the the root path "/". You can request on the root path, by not writing an endpoint):

    app.get("/");

  3. Add a callback as the second argument.
    The callback should recieve the HTTP request object "req" and the HTTP response object "res" as arguments:

    app.get("/", (req, res) => {

    });

  4. Send something in the response by using a method on the response object:
    • To send a file, use the sendFile method on the response object and pass the file path as parameter
      (The global variable "__dirname" is the path (name) for the directory of the file executing the code):

      app.get("/", (req, res) => {
          res.sendFile(__dirname + "/public/index.html");
      });


    • You can also use the send method to send the value of a variable.
      (Here an object is sent using the send method):

      app.get("/", (req, res) => {
          res.send({name: "James Smith", age: 22});
      });


      Objects will be converted to json format.

If you made a route for a get method on the server, you can now use your browser to get what the server sends, by going to the url of the route.
If you made a route for a get method on the router, you you can do the same if you use the router on the sever.
See how to use a router on a server here: Router.
(If you used the root path for the route, the endpoint of the route is just the domain of the server fx. "localhost:3000").
