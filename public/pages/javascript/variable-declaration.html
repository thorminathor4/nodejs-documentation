JavaScript - Variable Declaration

In JavaScript there are different ways to declare a variable.
Here are some differences between declaring variables using the three keywords.

const

Declares and defines a block-scoped variable, that must be initialized when declared and can not be reassigned.
If the variable is initialized to an object/array, the properties/values inside the object/array can be manipulated including reassignment.
Using the variable before it's initialisation will throw a ReffernceError.
When declared in an inner scope, with the same name as a variable in the outer scope, the variable from the outer scope can not be accessed in that inner scope.
Should always be used, when reassignment is not nessesary.

let

Declares and defines a block-scoped variable, that can be reassigned and where no initialization is required.
If declared without initialisation, its value will be initialized to undefined until assigned to another value.
Using the variable before it's initialisation will throw a ReffernceError.
When declared in an inner scope, with the same name as a variable in the outer scope, the variable from the outer scope can not be accessed in that inner scope.
Should only be used, when either:

  • It can not be initialized when it is declared.
  • Reassignment is nessesary.

var

AVOID USING THIS!
Declares a hoisted variable, that can be reassigned with it's where no initialization is required.
Hoisting means the variable is declared and initialized to undefinded in the start of it's scope.
When used inside a function, it declares a local variable scoped to the function.
If not declared inside a function, it declares a global variable.
When declared in a function, with the variable name being the same as a global variable, the global variable can not be accessed in that function.
It also permits redeclaration of variables.


You can also declare non hoisted global variables by initializing them without a keyword.
This is only possible, if the variable is not already defined.
