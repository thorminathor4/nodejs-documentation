JavaScript - Type Coercion

Type coercion is when values are converted automatically.
Coercion happens in specific cases using certain operation.
(There is not an explantion for when and how each operation coerces the values, because that requires a considerably long explanation).


To avoid coercion, do the following:
  • Convert values explicitly
  • Avoid using operations that can coerce values if possible

Non Coercing Operators

Strict (in)equality (===, !==)

Coercing Operators

Below are some operations that can coerce values:

Concationation (+)

The + operator can coerce values into strings and concatenate them.

Addition (+)

The + operator can coerce values into numbers (or *NaN) and add them.

Math operators (-, *, /, %)

Math operators can coerce values into numbers to perform the math operation.

Loose (in)equality (==, !=)

The loose (in)equality operator can coerce values which allows for loose (in)equality, which means:
  • Matching types is not required for loose equality.
  • Non matching types does not always result in loose inequality.

Relational operators (>, >=, <, <=)

Relational operators can coerce values into numbers (or *NaN) and compare them.

*NaN means "Not a Number", but is actually has the type Number.
