REST API

A REST API is a Web API that support CRUD operations for a ressource type.
REST is an abbreviation for Representational State Transfer.
A REST API has a state which it can transfer representations of.
A client can request a repressentation of the state, that a REST API is in, so it transfers the repressentation to the client.
A client can also transfer a repressentation of a state to the server, it wants the REST API to be in.
CRUD operations are the following operations: Create, Read, Update and Delete.

A REST API should meet the following requirements:
  • The ressource type should be a noun.
  • The endpoints should use the ressource type as a plural noun.
  • Each ressource should have a unique id.
  • A unique id should be assigned to each ressource upon creation.
  • The id's of the ressources should never be changed.
The table below shows the following:
  • Which operations the REST API should be able to perform.
  • Which HTTP method maps to which CRUD operation.
  • What the endpoint of each operation should be.
    (id should be a path variable).

(This is how a REST API for users should be defined):

Method Endoint Action
GET /users Reads all users
GET /users/id Reads one user by id
POST /users Creates one user
PUT /users/id Updates one user completely by id
PATCH /users/id Updates one user partially by id
DELETE /users/id Deletes one user by id
